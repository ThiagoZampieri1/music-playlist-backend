Documentação do Webservice

Bem-vindo à documentação da API REST para o sistema de gerenciamento de playlists. Esta API permite que você gerencie usuários, playlists e músicas através de requisições HTTP.

Autenticação

POST /auth.php?action=login

Realiza o login e retorna um token JWT para autenticação.

Parâmetros (JSON Body):

  • email (string) - Obrigatório
  • senha (string) - Obrigatório

Exemplo de Requisição:

{
  "email": "usuario@example.com",
  "senha": "sua_senha_segura"
}

Rotas para Usuários

POST /usuarios.php

Cadastra um novo usuário.

Parâmetros (JSON Body):

  • nome (string) - Obrigatório
  • email (string) - Obrigatório
  • senha (string) - Obrigatório

GET /usuarios.php

Lista todos os usuários cadastrados (requer autenticação).

GET /usuarios.php?id=ID

Obtém os detalhes de um usuário específico (requer autenticação).

PUT /usuarios.php?id=ID

Atualiza os dados de um usuário específico (requer autenticação).

DELETE /usuarios.php?id=ID

Exclui um usuário do sistema (requer autenticação).

Rotas para Playlists

GET /playlist.php

Lista todas as playlists do usuário autenticado.

POST /playlist.php

Cadastra uma nova playlist.

Parâmetros (JSON Body):

  • titulo (string) - Obrigatório
  • descricao (string) - Opcional

DELETE /playlist.php?id=ID

Exclui uma playlist específica do usuário autenticado.

Rotas para Músicas

POST /musica.php

Adiciona uma música a uma playlist.

Parâmetros (JSON Body):

  • titulo (string) - Obrigatório
  • artista (string) - Opcional
  • link (string) - Obrigatório
  • playlist_id (int) - Obrigatório

GET /musica.php?playlist_id=ID

Lista todas as músicas de uma playlist específica.

PUT /musica.php?musica_id=ID

Edita os dados de uma música específica.

Configuração do Ambiente

Para rodar o backend corretamente, siga os passos abaixo:

  1. Instale as dependências com composer install.
  2. Configure o banco de dados no arquivo .env.
  3. Rode o servidor com php -S localhost:8000.
