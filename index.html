Webservice REST - Sistema de Playlists

Bem-vindo à documentação da API REST para o sistema de gerenciamento de playlists. Esta API permite que você gerencie usuários, playlists e músicas através de requisições HTTP.

Rotas para Usuários

GET /usuarios.php

Lista todos os usuários cadastrados.

GET /usuarios.php?id=ID

Obtém os detalhes de um usuário específico.

POST /usuarios.php

Cadastra um novo usuário.

Parâmetros (JSON body):

  • nome (string) - Obrigatório
  • email (string) - Obrigatório
  • senha (string) - Obrigatório

PUT /usuarios.php?id=ID

Atualiza os dados de um usuário existente.

Parâmetros (JSON body):

  • nome (string) - Obrigatório
  • email (string) - Obrigatório

DELETE /usuarios.php?id=ID

Exclui um usuário do sistema.

Rotas para Playlists

GET /playlists.php

Lista todas as playlists disponíveis.

GET /playlists.php?usuario_id=ID

Lista as playlists de um usuário específico.

POST /playlists.php

Cadastra uma nova playlist.

Parâmetros (JSON body):

  • titulo (string) - Obrigatório
  • descricao (string) - Opcional
  • usuario_id (int) - Obrigatório

PUT /playlists.php?id=ID

Atualiza os dados de uma playlist existente.

Parâmetros (JSON body):

  • titulo (string) - Obrigatório
  • descricao (string) - Opcional

DELETE /playlists.php?id=ID

Exclui uma playlist do sistema.

Rotas para Músicas

GET /musicas.php?playlist_id=ID

Lista todas as músicas de uma playlist específica.

POST /musicas.php

Adiciona uma música a uma playlist.

Parâmetros (JSON body):

  • titulo (string) - Obrigatório
  • artista (string) - Opcional
  • link (string) - Obrigatório
  • plataforma (string) - Opcional
  • playlist_id (int) - Obrigatório

DELETE /musicas.php?playlist_id=ID&musica_id=ID

Remove uma música de uma playlist específica.

Observações

  • Todas as requisições devem ser feitas com o header Content-Type: application/json quando necessário.
  • Os endpoints suportam os métodos HTTP indicados em cada rota.
  • As respostas são retornadas no formato JSON.
  • Códigos de status HTTP são utilizados para indicar o resultado das operações:
    • 200 OK - Operação bem-sucedida.
    • 201 Created - Recurso criado com sucesso.
    • 400 Bad Request - Requisição inválida ou parâmetros faltando.
    • 404 Not Found - Recurso não encontrado.
    • 409 Conflict - Conflito de dados, como duplicidade.
    • 500 Internal Server Error - Erro no servidor.

Exemplos de Requisições

Criar um Novo Usuário

Método: POST

URL: /usuarios.php

Body:

{
    "nome": "Pedro Santos",
    "email": "pedro.santos@example.com",
    "senha": "minha_senha_segura"
}

Listar Playlists de um Usuário

Método: GET

URL: /playlists.php?usuario_id=1

Adicionar Música a uma Playlist

Método: POST

URL: /musicas.php

Body:

{
    "titulo": "Lose Yourself",
    "artista": "Eminem",
    "link": "https://example.com/loseyourself",
    "plataforma": "Spotify",
    "playlist_id": 2
}

Remover Música de uma Playlist

Método: DELETE

URL: /musicas.php?playlist_id=2&musica_id=5
